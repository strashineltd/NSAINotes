# NSAI笔记 3.0 — AI能力深度升级 设计规格

**Date**: 2026-05-09
**Status**: approved

## Overview

在 v2.6 基础上新增四个本地优先的 AI 子系统：RAG 知识库、长期记忆、AI Agent 工具链、多模态输入。

**架构决策**：方案A（全集成一体化），四个子系统直接集成进现有 Clean Architecture。
**计算架构**：纯本地优先。嵌入、向量检索、OCR 全部在设备端运行。AI 推理复用现有云端 Provider。

## Shared Infrastructure

四个子系统共享的基础组件：

| Component | Technology | Used by |
|-----------|-----------|---------|
| EmbeddingEngine | ONNX Runtime + all-MiniLM-L6-v2 (384-dim, ~23MB) | RAG, Memory |
| VectorStore | SQLite vector table + cosine similarity brute-force | RAG, Memory |
| ToolRegistry | Hilt @IntoSet multi-binding | Agent |
| OCREngine | Google MLKit Text Recognition v16 (offline) | Multimodal |

## Subsystem 1: RAG Knowledge Base

### Data Flow

```
Note saved → TextChunker → EmbeddingEngine → VectorStore
User query → EmbeddingEngine → VectorStore top-K → inject into AI context
```

### Components

**TextChunker** — Splits note content into 500-char chunks with 50-char overlap. Prefers splitting at paragraph boundaries (\\n\\n), then sentence boundaries.

**VectorStore** — SQLite `chunks` table:
```sql
CREATE TABLE chunks (
    id INTEGER PRIMARY KEY,
    note_id INTEGER NOT NULL,
    chunk_index INTEGER NOT NULL,
    content TEXT NOT NULL,
    embedding BLOB NOT NULL,  -- 384 float32 serialized (~1.5KB/row)
    created_at INTEGER NOT NULL
);
```
Search: brute-force cosine similarity. For <10,000 notes, full scan on 384-dim vectors is 5-50ms — no need for approximate nearest neighbor indices.

**IndexNotesUseCase** — Re-indexes a single note (delete old chunks, chunk, embed, insert). Triggered after note create/update in NoteRepositoryImpl.

**RetrieveContextUseCase** — Embeds query, searches VectorStore for top-K (default 5), filters by minSimilarity (0.6), returns concatenated context string for AI system prompt.

### Incremental Indexing

- Note create/update → immediate re-index
- Note delete → delete chunks
- "Rebuild all" button → clear vector table, re-index all notes (background coroutine)

### UI Changes

- AIHomeScreen: new "知识库搜索" mode button alongside Quick/Think/Agent
- KnowledgeBaseScreen (new): index status (indexed/total notes), rebuild button
- AI Chat: knowledge base mode automatically runs RAG retrieval before sending to AI

## Subsystem 2: Long-term Memory

### Memory Types

- PREFERENCE: "用户喜欢简洁的回答风格"
- FACT: "用户是一名Android开发者"
- CONVERSATION: "上次讨论了RAG架构方案"
- DATE: "项目截止日期是6月30日"
- RELATIONSHIP: "笔记#42 与 笔记#87 内容相关"

### Database

```sql
CREATE TABLE memories (
    id INTEGER PRIMARY KEY,
    type TEXT NOT NULL,
    key TEXT NOT NULL,
    content TEXT NOT NULL,
    importance REAL NOT NULL DEFAULT 0.5,
    embedding BLOB NOT NULL,
    source_conversation_id INTEGER,
    access_count INTEGER NOT NULL DEFAULT 0,
    last_accessed_at INTEGER NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (source_conversation_id) REFERENCES conversations(id)
);
```
Memory content encrypted via KeyStoreManager.

### Three Core Flows

**Extract** — After each conversation ends, AI extracts structured memories as JSON:
```
{messages → AI prompt → JSON: { memories: [{ type, key, content }] } }
```
New memories get embedded and stored. Duplicates (cosine >0.85) are merged instead.

**Retrieve** — At conversation start, embed current query, search memories table by cosine similarity, weight by importance × similarity. Top-K memories injected into system prompt.

**Consolidate** — WorkManager daily task: merge near-duplicate memories (cosine >0.9), archive memories unused for 30 days with importance <0.3.

### UI

- MemoryViewScreen (new): memory list grouped by type, swipe-to-delete, semantic search bar
- Settings: new "AI 记忆管理" entry
- Injection: memories are silently added to system prompt, not shown inline

## Subsystem 3: AI Agent Toolchain

### Architecture

```
User command → ReActLoop (Think → Act → Observe, max 8 steps)
                    ↓
             ToolRegistry (@IntoSet multi-binding)
                    ↓
      SearchNotes / CreateNote / WebSearch / SetReminder / ...
```

### AgentTool Interface

```kotlin
interface AgentTool {
    val name: String              // "search_notes"
    val displayName: String       // "搜索笔记"
    val description: String       // Tool description for AI
    val parameters: List<ToolParameter>
    val isDestructive: Boolean    // Requires user confirmation?
    suspend fun execute(params: Map<String, String>): ToolResult
}
```

### ReAct Loop

Each step: AI receives goal + available tools + history → returns JSON:
```
{ "thought": "...", "action": "tool_name", "params": {...} }
OR
{ "thought": "...", "final_answer": "..." }
```
If action: check permissions → execute → feed result back.
If final_answer: return to user. If max steps (8) or timeout: graceful failure.

### Built-in Tools (Batch 1)

| Tool | Destructive |
|------|-------------|
| search_notes | No |
| create_note | No |
| update_note | Yes |
| delete_note | Yes |
| list_notes | No |
| web_search | No |
| get_note_content | No |
| set_reminder | No |

### Safety

- Destructive operations: confirmation dialog with tool name + params
- User can choose "拒绝" / "仅本次" / "本次对话信任"
- All tool calls logged in conversation history for auditability
- "停止Agent" button available at all times

### UI

- Enhance existing Agent mode in AIHomeScreen
- Each Think/Act/Observe step shown as collapsible card
- Tool result: green checkmark (success) or red error icon
- Agent progress indicator showing current step

## Subsystem 4: Multimodal Input

### Input Modes

NoteEditScreen toolbar gets a camera icon → bottom sheet:
- 拍照识别文字 (OCR, purely local via MLKit)
- 从相册选择 (OCR, purely local via MLKit)
- AI 描述图片内容 (local OCR + optional AI vision call)
- 扫描文档 (MLKit Document Scanner)

### OCREngine

Wraps MLKit TextRecognition with DEFAULT_OPTIONS (Latin + CJK). Returns structured OCRResult with fullText and per-block confidence scores.

### DescribeImageUseCase

Two-step pipeline:
1. Local OCR via OCREngine
2. Compress image to Base64 (1024px max, JPEG quality 70%, output <100KB)
3. Build prompt with OCR text + image, send to vision-capable AI Provider

### AIProvider Vision Extension

New `supportsVision: Boolean` property on AIProvider enum. GLM (GLM-4V) and Qwen (Qwen-VL) support vision. AIOptions gets new `imageData: String?` field.

### Privacy

- OCR: 100% local, no data leaves the device
- AI vision mode: confirmation dialog before sending ("将发送图片到 [Provider] 进行分析")
- Compressed images typically <100KB

## Edge Cases

- **First launch after upgrade**: Embedding model extracted from assets (~23MB). Show "正在准备AI引擎..." progress. Chunking/indexing starts after model ready.
- **Empty notes**: Skip indexing for notes with empty content.
- **Private notes**: Decrypt before chunking/embedding. Re-encrypt after. Embeddings stored unencrypted (vectors cannot be reversed to original text).
- **Memory extraction failure**: AI service unavailable → skip extraction, retry next conversation. Graceful degradation.
- **Agent infinite loop**: Hard cap at 8 steps. Tool execution timeout 30s per step.
- **Model download failure (MLKit)**: Fallback to "需要联网下载OCR模型" message with retry button.
- **Storage pressure**: Embedding vectors ~1.5KB per chunk. 1000 notes × 5 chunks = ~7.5MB. Acceptable.
- **ONNX model update**: Versioned model file in assets. Check version on launch.
